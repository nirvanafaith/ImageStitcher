# 图片拼接软件 Spec

## 技术报告（设计分析）

### 1. 需求理解

构建一个 C++ 桌面图片拼接软件，用于将同一 `pageCode` 下的两张图片合并为一张，并同步修正源 XML 描述文件。典型场景是图书数字化校对流程中，将误切分的两张图片合并还原。

通过对 `d:\pinjie` 下范例文件的对比分析得出以下规则：

| 项目 | 原 XML (`2019年度...xml`) | 修改后 XML (`修改后.xml`) |
| --- | --- | --- |
| img00024001.jpg | 38×38 | 38×38（不变） |
| img00024002.jpg | 271×15 | **271×324**（高 15→324） |
| img00024003.jpg | 240×309 | **整段 `<段落>` 被删除** |
| img00024004.jpg | 358×253 | 358×253（不变） |

**推导出的拼接规则**（上下拼接，img00024002 在上、img00024003 在下）：
- 新宽度 = `max(w1, w2)` = `max(271, 240)` = **271**
- 新高度 = `h1 + h2` = `15 + 309` = **324**
- 输出文件名 = 小序号图（img00024002.jpg）
- 删除 = 大序号图文件（img00024003.jpg）
- XML 变更：更新小序号图 `<插图>` 内的 `<图片宽度>/<图片高度>`；删除大序号图所在的整个 `<段落>` 块（含 `<插图>` 与外层 `<段落>`）。

### 2. 关键设计决策

**(A) 对齐语义统一化**：`居左/中线/居右` 在两种拼接方向下指代"垂直于拼接方向的轴"上的对齐：
- 上下拼接 → 对齐的是水平方向：居左=窄图靠左、中线=居中、居右=靠右
- 左右拼接 → 对齐的是垂直方向：居左=矮图靠上、中线=居中、居右=靠下
- 这样 6 种组合共用一组对齐术语，UI 一致、符合人体工学。UI 上以 tooltip 说明。

**(B) 拼接位置顺序**：用户输入的"图片1"=上/左，"图片2"=下/右。输出命名/删除与小序号绑定（与输入顺序无关），始终保留小序号名、删除大序号文件。

**(C) 空白填充**：JPG 不支持透明，画布空白区域用**白色**填充（符合印刷出版物场景）。

**(D) 技术选型**：**Qt 5.15.2 LTS + QImage + QDomDocument**（**为兼容 Win7+ 必须使用 Qt 5.15，因 Qt 6 已不支持 Win7**）。单一框架覆盖 GUI/图像/XML/文件 IO，依赖少、部署简、性能足够。无需 OpenCV（本任务为几何拼接，非特征匹配拼接）。XML 用 DOM 模型，符合"原地修改节点值 + 删除整段"语义。Qt 5.15 与 Qt6 在本项目所用 API（QImage/QPainter/QDomDocument/QWidget）上基本一致，迁移成本低。

**(E) "不生成新 xml" 的实现**：XML 不能字节级原地增删，唯一可行方式是读入内存→修改 DOM 树→覆盖写回原路径。此为"在源 XML 上直接修改"的标准实现，不产生额外 xml 文件。

### 3. 架构（高内聚低耦合分层）

```
UI 层 (MainWindow)
    ├── 拼接服务 (Stitcher)        ── 纯算法，输入 QImage×2 + 模式，输出 QImage
    ├── XML 服务 (XmlEditor)       ── QDomDocument 加载/改/删/写回
    ├── 文件服务 (FileManager)     ── 图片加载/保存/删除 + 序号解析
    └── 序号工具 (NumberUtil)      ── 文件名→数值序号
```

依赖方向单向：UI → 各服务；服务层之间互不依赖，可独立测试与替换。

### 4. 拼接算法（性能：微秒级，无需优化）

```
上下拼接:
  newW = max(w1, w2);  newH = h1 + h2
  canvas(newW, newH) 填白
  x1 = alignOffset(w1, newW);  drawImage(x1, 0, img1)
  x2 = alignOffset(w2, newW);  drawImage(x2, h1, img2)

左右拼接:
  newW = w1 + w2;  newH = max(h1, h2)
  canvas(newW, newH) 填白
  y1 = alignOffset(h1, newH);  drawImage(0, y1, img1)
  y2 = alignOffset(h2, newH);  drawImage(w1, y2, img2)

alignOffset(size, total):
  居左=0; 中线=(total-size)/2; 居右=total-size
```

图片格式统一转 `QImage::Format_RGB32`（JPG 无 alpha），避免色偏。JPG 保存质量 95。

### 5. UI（极简 + 人体工学）

- 左侧控制面板（固定宽 ~360px）：文件夹路径(+浏览) / XML 路径(+浏览) / 图片1名 / 图片2名 / 拼接方向(上下·左右) / 对齐(居左·中线·居右) / 确认预览 / 输出
- 右侧展示区（Expanding）：`Qt::KeepAspectRatio` + `SmoothTransformation` 缩放显示，居中、不越界、最大化
- 输出按钮在未预览前禁用；输出前弹确认对话框（破坏性操作）
- 白底浅灰边、单一主色按钮、留白适当、无装饰

### 6. 数据库范式说明

本项目无数据库，仅有 XML 文件结构。XML 中每个 `<插图>` 仅描述单张图片属性（链接、宽、高），无传递依赖、无部分依赖，符合 BC 范式精神。

### 7. 平台兼容性（Win7+）

- **Qt 版本**：Qt 5.15.2 LTS（Qt 6 不支持 Win7，必须用 5.15）。
- **编译器**：MSVC 2019 64-bit（v142 工具集），生成 Win7 兼容代码。
- **宏定义**：`_WIN32_WINNT=0x0601`、`WINVER=0x0601`（声明目标 Win7）。
- **运行时**：静态链接 VC 运行库（/MT）或随包分发 vcredist，避免目标机缺 DLL。
- **部署**：`windeployqt --release --compiler-runtime` 自动打包 Qt5Core/Gui/Widgets/Xml.dll、platforms/qwindows.dll、imageformats/qjpeg.dll。目标机无需预装 Qt。
- **验证**：在 Win7 SP1 真机/虚拟机上验证启动、预览、输出全流程。

### 8. 效率优化策略

- **图片并行加载**：用 `QtConcurrent::run` 并行加载两张图片，IO+解码时间近减半。
- **预览缩放缓存**：拼接后的 QImage 缓存为成员变量，窗口 `resizeEvent` 仅对缓存 `scaled()`，不重新拼接原图。
- **格式一次性转换**：`loadImage` 时统一转 `Format_RGB32`，后续不再做格式转换。
- **JPG 保存质量 95**：兼顾质量与编码速度。
- **XML 单次内存修改**：QDomDocument 一次性加载→改→写回，无反复 IO。
- **拼接算法**：QPainter 由 Qt 底层优化，单次拼接微秒级，无需手写 SIMD。

---

## Why

需要一个 C++ 工具将误切分的两张图片合并为一张并同步修正 XML 描述，避免手工用图片编辑器拼接 + 手工改 XML 的低效与易错。

## What Changes

- 新增 C++/Qt 5.15.2 LTS 图片拼接桌面应用（兼容 Win7 SP1 及以后系统）
- 实现 6 种拼接组合（2 方向 × 3 对齐）的内存预览
- 实现输出流程：保存拼接图覆盖小序号文件 + 删除大序号文件 + 原地修改源 XML（更新宽高 + 删除大序号段落）
- 极简人体工学 UI，右侧预览区按比例最大化显示
- 效率优化：图片并行加载、预览缩放缓存、格式一次性转换

## Impact

- Affected code: 全新项目，位于 `d:\pinjie\src\` 子目录
- 依赖：Qt 5.15.2 LTS (Widgets / Gui / Core / Xml)，MSVC 2019 64-bit，CMake 3.16+ 构建
- 平台：Windows 7 SP1 及以后，64-bit
- 部署：windeployqt 打包，目标机无需预装 Qt
- 不影响现有范例 XML/图片文件（输出操作由用户主动触发）

## ADDED Requirements

### Requirement: 图片拼接预览

系统 SHALL 提供界面录入图片文件夹路径、XML 文件路径、两个图片文件名，选择拼接方向（上下/左右）与对齐方式（居左/中线/居右），点击确认后在右侧展示区显示按比例最大化不越界的拼接预览图。

#### Scenario: 正常预览
- **WHEN** 用户填入有效文件夹、XML、两个存在的图片名，选择上下拼接+居中对齐，点击确认
- **THEN** 右侧展示区显示两图上下堆叠、窄图居中、白色填充空白、整体按展示区比例最大化显示

#### Scenario: 输入校验
- **WHEN** 文件夹/XML/图片任一不存在，或两图序号相同
- **THEN** 弹错误提示，不进行预览

### Requirement: 拼接输出与 XML 修正

系统 SHALL 在点击输出时：将拼接图保存为小序号图片名（覆盖原文件）；删除大序号图片文件；在源 XML 上原地修改——更新小序号 `<插图>` 的 `<图片宽度>/<图片高度>` 为拼接后尺寸，删除大序号 `<插图>` 所在的整个 `<段落>` 块；不产生新 XML 文件。

#### Scenario: 上下拼接输出（范例场景）
- **GIVEN** 图片1=img00024002.jpg(271×15)、图片2=img00024003.jpg(240×309)，上下拼接
- **WHEN** 点击输出并确认
- **THEN** imageCut\img00024002.jpg 被替换为 271×324 的拼接图；imageCut\img00024003.jpg 被删除；源 XML 中 img00024002 的宽高改为 271/324，img00024003 所在 `<段落>` 被删除；无新 XML 文件产生

#### Scenario: 左右拼接输出
- **GIVEN** 两图左右拼接
- **THEN** 拼接图宽=w1+w2、高=max(h1,h2)，按对齐放置矮图；命名/删除/XML 修正逻辑同上

#### Scenario: 破坏性操作确认
- **WHEN** 点击输出
- **THEN** 弹出确认对话框，用户确认后才执行文件与 XML 修改

### Requirement: 拼接尺寸计算

系统 SHALL 按以下规则计算拼接后尺寸：
- 上下拼接：宽=max(w1,w2)，高=h1+h2
- 左右拼接：宽=w1+w2，高=max(h1,h2)

### Requirement: 序号解析

系统 SHALL 从图片文件名（形如 `img00024002.jpg`）解析数值序号用于大小比较，输出保留小序号名、删除大序号文件。

#### Scenario: 序号比较
- **GIVEN** img00024002.jpg 与 img00024003.jpg
- **THEN** 解析得 24002 与 24003，保留 img00024002.jpg，删除 img00024003.jpg

### Requirement: Windows 7 及以后系统兼容

系统 SHALL 基于 Qt 5.15.2 LTS 与 MSVC 2019 64-bit 构建，定义 `_WIN32_WINNT=0x0601`/`WINVER=0x0601`，生成的可执行文件 SHALL 能在 Windows 7 SP1 及以后版本上启动并完整运行预览与输出流程；SHALL 通过 windeployqt 打包所需 Qt DLL 与插件，目标机无需预装 Qt。

#### Scenario: Win7 真机运行
- **WHEN** 在 Win7 SP1 机器上运行打包后的 exe
- **THEN** 程序正常启动，能完成预览与输出，无缺 DLL 报错

### Requirement: 运行效率

系统 SHALL 对性能敏感路径做优化：图片加载并行化（QtConcurrent 并行加载两图）、预览缩放缓存（resize 时不重新拼接）、图片格式一次性转换、JPG 保存质量 95。

#### Scenario: 大图预览响应
- **WHEN** 加载两张较大图片并预览
- **THEN** 首次预览在合理时间内完成；窗口缩放时预览刷新不重新拼接原图
